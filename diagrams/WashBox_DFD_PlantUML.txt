PLANTUML CODE FOR DRAW.IO - WashBox Data Flow Diagram
=====================================================

@startuml WashBox_DFD
!define ENTITY_COLOR #E1F5FE
!define PROCESS_COLOR #FFF9C4
!define DATASTORE_COLOR #F0F4C3

skinparam linetype ortho
skinparam backgroundColor #FFFFFF

' External Entities
rectangle "EXTERNAL ENTITIES" #FFFFFF {
    actor "Customer\nApp" as CUST #ENTITY_COLOR
    actor "Admin\nPanel" as ADMIN #ENTITY_COLOR
    actor "Branch\nStaff" as BRANCH #ENTITY_COLOR
    actor "Driver\nApp" as DRIVER #ENTITY_COLOR
    actor "Firebase\nFCM" as FCM #ENTITY_COLOR
    actor "GCash\nAPI" as GCASH #ENTITY_COLOR
    actor "Email\nService" as EMAIL #ENTITY_COLOR
}

' Core Processes
rectangle "CORE PROCESSES" #FFFFFF {
    component "P1\nAuthentication\n& Profile" as P1 #FFF9C4
    component "P2\nService &\nPromotion" as P2 #FFF9C4
    component "P3\nPickup Request\nMgmt" as P3 #FFF9C4
    component "P4\nLaundry Order\nProcessing" as P4 #FFF9C4
    component "P5\nPayment\nVerification" as P5 #FFF9C4
    component "P6\nInventory\nMgmt" as P6 #FFF9C4
    component "P7\nLocation\nTracking" as P7 #FFF9C4
    component "P8\nStatus &\nNotifications" as P8 #FFF9C4
    component "P9\nFinancial\nTracking" as P9 #FFF9C4
    component "P10\nReporting &\nAnalytics" as P10 #FFF9C4
    component "P11\nDevice Token\nMgmt" as P11 #FFF9C4
    component "P12\nAudit\nLogging" as P12 #FFF9C4
}

' Data Stores
rectangle "DATA STORES" #FFFFFF {
    database "D1\nCustomer\nAccounts\n15K records" as D1 #F0F4C3
    database "D2\nServices &\nCategories\n300 records" as D2 #F0F4C3
    database "D3\nPickup\nRequests\n50K records" as D3 #F0F4C3
    database "D4\nLaundry\nOrders\n100K records" as D4 #F0F4C3
    database "D5\nPayment\nRecords\n80K records" as D5 #F0F4C3
    database "D6\nInventory\nItems\n5K records" as D6 #F0F4C3
    database "D7\nLocation\nHistory\n1M records" as D7 #F0F4C3
    database "D8\nPromotions\n500 records" as D8 #F0F4C3
    database "D9\nDevice\nTokens\n20K records" as D9 #F0F4C3
    database "D10\nFinancial\nTx\n200K records" as D10 #F0F4C3
    database "D11\nAudit\nLogs\n1M records" as D11 #F0F4C3
}

' DATA FLOWS - EXTERNAL TO PROCESS
CUST --> P1 : login, profile
CUST --> P3 : request pickup
CUST --> P4 : submit laundry
CUST --> P5 : payment proof

ADMIN --> P2 : manage services
ADMIN --> P5 : verify payments
ADMIN --> P10 : view reports

BRANCH --> P3 : accept pickup
BRANCH --> P4 : process laundry
BRANCH --> P6 : manage inventory

DRIVER --> P3 : accept pickup
DRIVER --> P7 : update location
DRIVER --> P4 : deliver laundry

FCM --> P8 : notifications
P11 --> FCM : token data

GCASH --> P5 : payment status
P9 --> GCASH : payment request

EMAIL --> P8 : send email
P8 --> EMAIL : email data

' PROCESS TO DATASTORE FLOWS
P1 <--> D1 : read/write customer
P2 <--> D2 : read/write services
P2 <--> D8 : read/write promos

P3 <--> D3 : read/write pickups
P3 --> D1 : read customer
P3 <--> D1 : customer info

P4 <--> D4 : read/write orders
P4 --> D6 : read inventory
P4 --> D3 : read pickup
P4 --> D2 : read services

P5 <--> D5 : read/write payments
P5 --> D4 : read laundry amount
P5 --> D1 : read customer

P6 <--> D6 : read/write stock
P6 --> D4 : read orders

P7 <--> D7 : store locations
P7 --> D3 : read pickup

P8 --> D9 : read tokens
P8 --> D3 : read pickup status
P8 --> D4 : read order status
P8 --> D5 : read payment status

P9 <--> D10 : record transactions
P9 --> D5 : read payments

P10 --> D1 : read customers
P10 --> D3 : read pickups
P10 --> D4 : read orders
P10 --> D5 : read payments
P10 --> D10 : read transactions

P11 <--> D9 : store tokens

P12 <--> D11 : log activities
P12 --> D1 : log user actions
P12 --> D3 : log pickup changes
P12 --> D5 : log payment changes

' INTER-PROCESS FLOWS
P1 -.-> P11 : customer ID
P3 -.-> P4 : pickup completed
P4 -.-> P5 : order completed, amount
P5 -.-> P9 : payment confirmed
P3 -.-> P8 : status update
P4 -.-> P8 : status update
P5 -.-> P8 : status update
P7 -.-> P8 : location update
P4 -.-> P6 : stock usage

@enduml

INSTRUCTIONS FOR DRAW.IO
========================

1. Go to https://draw.io
2. Create NEW DIAGRAM
3. Choose PlantUML from template OR use File > Import
4. Paste the above code
5. Click Convert to draw.io format
6. Edit and customize as needed

ALTERNATIVE METHOD - Copy XML:
==============================
1. Go to https://draw.io
2. Create NEW DIAGRAM
3. Edit > Edit Data (XML)
4. Paste XML code (provided separately)
5. Click Apply

KEY COMPONENTS
==============

External Entities (Blue): Customer App, Admin, Branch, Driver, Firebase, GCash, Email
Processes (Yellow): P1-P12 (12 processes)
Data Stores (Green): D1-D11 (11 databases)

CONNECTIONS:
- Solid lines: Direct data flows
- Dashed lines: Inter-process triggers
- All labeled with data content
